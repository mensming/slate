    Conenza Community Platform JSON Rest User API Developer Guide

    Introduction Audience

    This document is intended to provide instructions for developers writing to the Conenza User API. For specific questions, please contact your Conenza Account Manager.

    Conenza User API Basics The Conenza User API is a RESTful web service that supports requests to register, update, delete, and retrieve users on the Conenza Community Platform.

    You access these resources by sending an HTTP request to the Conenza API server. The server replies with a response that either contains the data you requested, id, or error message.

    You must supply username/password authentication information in the request using the HTTP BASIC authentication mechanism. See Authentication, below for more details.

    In these documents, when a portion of a URL, path, or parameter value is shown in angle brackets , this indicates that you should replace the bracketed value with a particular value appropriate to your request.

    Getting User Information User information can be retrieved using GET. Users can be identified with their user ID, email, or CSID.

    Action Path Method Request content-type Get info on the first 100 Users /api/v1/users GET N/A Get a page of Users /api/v1/users?start=1&end=10 GET N/A Get a single User’s Info with userId /api/v1/users/ GET N/A Get a single User’s Info with another identifier /api/v1/users// GET N/A Get a single User’s field /api/v1/users//?fieldList=, GET N/A

    All of the POST fields available during registration are also available with GET except image. Additionally, the following fields are read-only, available with GET but not modifiable using PUT or POST.

    Tag lastFailedLoginDate failedLoginAttempts Lockout lockoutDate createdDate createdBy modifiedDate modifiedBy lastLoginDate activationDate ssoUser loginDate

    Inactivating a User With DELETE, users are inactivated but not removed from the system.

    Action Path Method Request content-type Delete a User /api/v1/users/< userId> DELETE N/A

    Modifying a User Specific user fields can be modified using PUT. The same fields available for registration are also available to be modified.

    Action Path Method Request content-type Modify a User /api/v1/users/ PUT application/json

    Generic HTTP Response Codes 400 BAD_REQUEST The request you made failed validation. See the content body for a detailed error message.

    404 NOT_FOUND The request you made was for an unknown resource.

    500 INTERNAL_SERVER_ERROR Unanticipated error. Please contact Conenza Support and provide a copy of your request file and the reported error.

    501 NOT_IMPLEMENTED The request you made tried to use an HTTP method that is not supported by the API or an HTTP method that does not support your specific request (such as a non-EXPIRE status change).

    415 UNSUPPORTED MEDIA TYPE The request you made had the wrong MIME type.
